# คู่มือการใช้งาน Import ข้อมูลบุคลากร

**เส้นทาง:** `/police-personnel/import`

---

## ภาพรวม

ระบบ Import ใช้สำหรับนำเข้าข้อมูลบุคลากรตำรวจจากไฟล์ Excel (.xlsx / .xls) เข้าสู่ฐานข้อมูล รองรับ 2 โหมด:

| โหมด | วัตถุประสงค์ | จำนวนคอลัมน์ |
|------|-------------|--------------|
| **Import แบบเต็ม** | นำเข้าข้อมูลบุคลากรทั้งหมด | 24 คอลัมน์ |
| **อัปเดตผู้สนับสนุน** | อัปเดตเฉพาะ ตำแหน่งที่ร้องขอ / ผู้สนับสนุน / เหตุผล | 6 คอลัมน์ |

---

## โหมดที่ 1: Import แบบเต็ม

### ขั้นตอน

**ขั้นที่ 1 — เลือกโหมดและปี**

1. ไปที่หน้า **นำเข้าข้อมูล Police Personnel**
2. คลิก **"Import แบบเต็ม (ไม่ลบข้อมูลเดิม)"**
3. เลือก **ปี พ.ศ.** ที่ต้องการนำเข้าจาก dropdown (default = ปีปัจจุบัน)

**ขั้นที่ 2 — ดาวน์โหลด Template**

1. คลิกปุ่ม **"ดาวน์โหลด Template"**
2. ระบบจะดาวน์โหลดไฟล์ `police_personnel_template.xlsx`
3. เปิดไฟล์ใน Excel — จะพบหัวคอลัมน์ 24 คอลัมน์ดังนี้:

| ลำดับ | ชื่อคอลัมน์ | หมายเหตุ |
|-------|------------|---------|
| 1 | อาวุโส | ลำดับอาวุโส |
| 2 | ยศ | ยศตำรวจ |
| 3 | ชื่อ สกุล | ชื่อ-นามสกุล (มีเว้นวรรค) |
| 4 | ID | รหัสประจำตัว |
| 5 | POSCODE | รหัส POS Code |
| 6 | ตำแหน่ง | ชื่อตำแหน่ง |
| 7 | เลขตำแหน่ง | เลขประจำตำแหน่ง |
| 8 | ทำหน้าที่ | รักษาการ/ปฏิบัติหน้าที่แทน |
| 9 | แต่งตั้งครั้งสุดท้าย | วันที่ (DD/MM/YYYY พ.ศ.) |
| 10 | ระดับนี้เมื่อ | วันที่ (DD/MM/YYYY พ.ศ.) |
| 11 | บรรจุ | วันที่บรรจุ (DD/MM/YYYY พ.ศ.) |
| 12 | วันเกิด | วันที่ (DD/MM/YYYY พ.ศ.) |
| 13 | คุณวุฒิ | วุฒิการศึกษา |
| 14 | เลขประจำตัวประชาชน | 13 หลัก (ว่างได้สำหรับตำแหน่งว่าง) |
| 15 | หน่วย | ชื่อหน่วยงาน |
| 16 | เกษียณ | วันที่เกษียณ (DD/MM/YYYY พ.ศ.) |
| 17 | จำนวนปี | จำนวนปีรับราชการ |
| 18 | อายุ | อายุ (ปี) |
| 19 | ตท. | สถานที่ฝึกอบรม |
| 20 | นรต. | หลักสูตรนักเรียนนายร้อยตำรวจ |
| 21 | หมายเหตุ/เงื่อนไข | หมายเหตุเพิ่มเติม |
| 22 | ตำแหน่งที่ร้องขอ | ตำแหน่งที่ต้องการย้าย |
| 23 | ชื่อผู้สนับสนุน | ชื่อผู้สนับสนุน |
| 24 | เหตุผล | เหตุผลที่ขอย้าย |

**ขั้นที่ 3 — กรอกข้อมูลใน Excel**

- กรอกข้อมูลตั้งแต่แถวที่ 2 เป็นต้นไป (แถวที่ 1 คือหัวคอลัมน์)
- **ห้ามลบหรือเปลี่ยนชื่อหัวคอลัมน์**
- วันที่ใช้รูปแบบ `DD/MM/YYYY` ปีเป็น พ.ศ. เช่น `01/01/2568`
- สำหรับ **ตำแหน่งว่าง** (ไม่มีบุคคลครอง): เว้น `เลขประจำตัวประชาชน` ว่างไว้

**ขั้นที่ 4 — อัปโหลดไฟล์**

1. บันทึกไฟล์ Excel
2. กลับไปที่หน้า Import ในระบบ
3. คลิกที่กรอบประกาย **"คลิกเพื่อเลือกไฟล์ Excel"** หรือลาก-วางไฟล์
4. ระบบจะแสดงชื่อไฟล์ที่เลือก
5. คลิกปุ่ม **"นำเข้าข้อมูล"**

**ขั้นที่ 5 — รอผลการประมวลผล**

- ระบบจะแสดง **Progress Bar** พร้อมเปอร์เซ็นต์และจำนวนรายการที่ประมวลผลแล้ว
- การ import ทำงานเป็น background job (polling ทุก 2 วินาที)
- ไม่ต้องรีเฟรชหน้าจอ รอจนกว่า progress จะถึง 100%

**ขั้นที่ 6 — ตรวจสอบผลลัพธ์**

หลัง import เสร็จ ระบบจะแสดงสรุปผล:
- 🟢 **นำเข้าสำเร็จ** — จำนวนแถวที่บันทึกได้
- 🔴 **ล้มเหลว** — จำนวนแถวที่มีข้อผิดพลาด
- หากมีข้อผิดพลาด ระบบจะแสดงตารางรายละเอียดแต่ละแถวที่ล้มเหลว พร้อมสาเหตุ

### พฤติกรรมของระบบ (Logic การบันทึก)

| สถานการณ์ | การกระทำ |
|-----------|---------|
| ไม่เคยมีข้อมูลปีนี้มาก่อน | INSERT ทั้งหมดด้วย `createMany` (เร็วที่สุด) |
| มีข้อมูลปีนี้อยู่แล้ว + มีเลขบัตรประชาชน | UPSERT ด้วย `nationalId + year` |
| มีข้อมูลปีนี้อยู่แล้ว + ไม่มีเลขบัตร + มีเลขตำแหน่ง | ค้นหาด้วย `positionNumber + year` แล้ว UPDATE |
| มีข้อมูลปีนี้อยู่แล้ว + ไม่มีเลขบัตรและเลขตำแหน่ง | INSERT ใหม่เสมอ |

## โหมดที่ 2: อัปเดตผู้สนับสนุน

ใช้เมื่อต้องการอัปเดตเฉพาะข้อมูล **ตำแหน่งที่ร้องขอ / ชื่อผู้สนับสนุน / เหตุผล** โดยไม่แตะข้อมูลอื่น

### ขั้นตอน

**ขั้นที่ 1 — เลือกโหมดและปี**

1. คลิก **"อัปเดตผู้สนับสนุนเท่านั้น (ไม่ลบข้อมูลเดิม)"**
2. เลือก **ปี พ.ศ.** ที่ต้องการอัปเดต (ต้องตรงกับปีที่ Import ไปแล้ว)

**ขั้นที่ 2 — ดาวน์โหลด Template**

1. คลิกปุ่ม **"ดาวน์โหลด Template"**
2. ระบบจะดาวน์โหลดไฟล์ `police_personnel_supporter_template.xlsx`
3. ไฟล์นี้มี **6 คอลัมน์** เท่านั้น:

| ลำดับ | ชื่อคอลัมน์ | จำเป็น | หมายเหตุ |
|-------|------------|--------|---------|
| 1 | ชื่อ สกุล | ✅ | ต้องตรงกับข้อมูลในระบบ |
| 2 | เลขตำแหน่ง | ✅ | ใช้ค้นหาข้อมูลในระบบ |
| 3 | เลขประจำตัวประชาชน | ⬜ | เว้นว่างได้ถ้าเป็นตำแหน่งว่าง |
| 4 | ตำแหน่งที่ร้องขอ | ⬜ | ตำแหน่งที่ต้องการย้าย |
| 5 | ชื่อผู้สนับสนุน | ⬜ | ชื่อผู้สนับสนุนการย้าย |
| 6 | เหตุผล | ⬜ | เหตุผลประกอบคำร้อง |

**ขั้นที่ 3 — กรอกข้อมูล**

- `ชื่อ สกุล` และ `เลขตำแหน่ง` ต้องกรอกทุกแถว
- สำหรับ **ตำแหน่งที่มีคน**: ใส่ `เลขประจำตัวประชาชน` เพื่อให้ค้นหาแม่นยำขึ้น
- สำหรับ **ตำแหน่งว่าง**: เว้น `เลขประจำตัวประชาชน` ว่าง ระบบจะใช้ `เลขตำแหน่ง` ค้นหา

**ขั้นที่ 4 — 6: เหมือน Import แบบเต็ม** (เลือกไฟล์ → คลิกนำเข้า → รอผล)

### ผลลัพธ์เพิ่มเติมสำหรับโหมดนี้

- 🟡 **ไม่พบในระบบ** — จำนวนแถวที่ค้นหาบุคลากรไม่พบ (ชื่อหรือเลขตำแหน่งไม่ตรง)
- หากอัปเดตสำเร็จไม่เกิน 20 รายการ จะแสดงตารางรายชื่อที่อัปเดตแล้วด้วย

## ข้อควรระวัง

> ⚠️ **สำคัญ: ห้ามลบหรือเปลี่ยนชื่อหัวคอลัมน์** เพราะระบบใช้ชื่อคอลัมน์ในการ map ข้อมูล

- ไฟล์ต้องเป็น `.xlsx` หรือ `.xls` เท่านั้น (ไม่รองรับ `.csv`)
- Import แบบเต็มต้องมีอย่างน้อย **10 คอลัมน์** (ระบบตรวจสอบอัตโนมัติ)
- อัปเดตผู้สนับสนุนต้องมี **ไม่เกิน 10 คอลัมน์** — ถ้าเกินระบบจะแจ้งว่าใช้ไฟล์ผิด template
- ต้องมีข้อมูล **POS Code Master** ในระบบก่อน Import แบบเต็มจึงจะสำเร็จ
- ระหว่าง import ห้ามปิดหน้าต่าง browser

## การแก้ไขข้อผิดพลาดที่พบบ่อย

| ข้อผิดพลาด | สาเหตุ | วิธีแก้ |
|-----------|--------|--------|
| `ไฟล์นี้มีเพียง X คอลัมน์` | ไฟล์ไม่ถูกต้อง | ดาวน์โหลด Template ใหม่แล้วกรอกข้อมูลซ้ำ |
| `กรุณาเพิ่มข้อมูล POS Code Master` | ยังไม่มี POS Code ในระบบ | เพิ่ม POS Code Master ก่อน |
| `ไม่พบข้อมูล "ชื่อ" เลขตำแหน่ง: XXXXX` | ชื่อหรือเลขตำแหน่งไม่ตรงกับฐานข้อมูล | ตรวจสอบข้อมูลในไฟล์ให้ตรงกับที่ Import ไปแล้ว |
| `ไม่มีคอลัมน์ที่จำเป็น: ชื่อ สกุล` | ใช้ไฟล์ template ผิดประเภท | ดาวน์โหลด Template ให้ตรงกับโหมดที่เลือก |
| แถวล้มเหลวทั้งหมด | ข้อมูลรูปแบบไม่ถูกต้อง | ตรวจสอบรูปแบบวันที่และตัวเลข |

## ลำดับการทำงานที่แนะนำ (ปีใหม่)

```
1. Import แบบเต็ม (ครั้งแรก)
   └── ดาวน์โหลด template → กรอกข้อมูลบุคลากรทั้งหมด → อัปโหลด

2. อัปเดตผู้สนับสนุน (ต่อมา)
   └── ดาวน์โหลด template supporter → กรอกเฉพาะคนที่มีผู้สนับสนุน → อัปโหลด

3. Import แบบเต็มซ้ำ (ถ้าข้อมูลเปลี่ยน)
   └── ระบบจะ UPSERT โดยอัตโนมัติ ไม่สร้างรายการซ้ำ
```
